## User Stories

### 1. Onboarding (MVP: simple structured intake)

As a job seeker, I want a simple, guided onboarding when I first sign up, so that I can set my preferences and start getting relevant recommendations right away.

Acceptance Criteria:

- LinkedIn O-Auth to log in
- LinkedIn profile and CV
	- User profile is pulled from LinkedIn in the background and the platform creates the candidate's brief (see below) seeded from this input
	- User should be able to upload their CV (word/pdf format) (optional step, but recommended - as such option can be done later, post onboarding)
- Preferences
	- The preferences screen captures two free-text inputs from the user:
    - **"What kind of role are you looking for?"**
	    - captures role function, seniority, company type, and day-to-day scope. Guided by three persistent sub-prompts (role & level, kind of company, day-to-day work) and a sample answer the user can expand for calibration.
    - **"Where won't you compromise?"** 
	    - captures hard filters / non-negotiables. Memory-jogger nudge chips (location, compensation, work hours / on-call, industry, company stage / size, role type, domain, travel) are available below the input to help users think across dimensions they might otherwise skip. A sample answer is available for calibration.
- On submission, the platform parses free-text inputs into structured preference dimensions using LLM extraction. Each extracted preference is tagged as either:
    - **Non-negotiable** (sourced from Q2) — used as hard filters in matching; never violated.
    - **Soft preference** (sourced from Q1) — used as ranking signals in matching.
- After extraction, the user is shown a parsed preference summary for confirmation before proceeding. This screen:
    - Displays what the system understood, organized by dimension.
    - Allows the user to correct any misinterpretation inline.
    - Surfaces any missing dimensions (e.g., if compensation was not mentioned, it prompts: "You didn't mention compensation — is that flexible?").
- User cannot proceed to the main app without completing both text inputs and confirming the parsed summary.
- On completion, the platform initializes the Preferences section of the Candidate Brief from these inputs (see below)
