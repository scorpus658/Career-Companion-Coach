
# Epic: Clarify Intent

[[B. Value chain with objectives]]

## Objective

Build a precise career profile that anchors every recommendation, and keep it current as priorities change.

## User Stories

### 1. Onboarding (MVP: simple structured intake)

As a job seeker, I want a simple, guided onboarding when I first sign up, so that I can set my core preferences quickly and start getting relevant recommendations right away.

Acceptance Criteria:

- LinkedIn O-Auth to log in
- Onboarding captures the core preference fields:
  - **Location preferences** — preferred location(s) and work mode (remote / hybrid / onsite).
  - **Compensation structure and expectation** — structure (ESOP / fixed / mix) and expected amount/range.
  - **Company stage** — preferred stage(s) of company (e.g., early-stage / growth / late-stage / public).
  - **Domain preferences** — e.g., Marketing, Product Management, Consulting, Sales, Operations.
  - **Role preferences** — e.g., Founder's Office, Chief of Staff, etc.
  - **Experience level**.
- Each field is a structured input (dropdowns, chips, ranges) — no free-text conversational interview in the MVP.
- User cannot proceed to the main app without completing the minimum required fields.
- On completion, the platform creates the candidate's living Candidate Brief (see below) seeded from these inputs.

Notes:
- Phase 2 (MLP): replace structured intake with a conversational onboarding that draws out unstated preferences.
- Platform: responsive web app (see Cross-cutting → Web App).

---

### 2. Candidate Brief (living candidate file)

As the platform, I want to maintain a living Candidate Brief — a single evolving record of everything we know about the candidate (stated preferences, profile facts, and behavioural signals over time) — so that every recommendation draws from the most current understanding of the user.

Acceptance Criteria:
- The Brief is initialized from onboarding inputs.
- The Brief is continuously updated from two sources in the MVP: (a) explicit user edits, and (b) behavioural signals from the daily stack (right/left swipes act as positive/negative preference signals — see Discover Roles).
- Any update to the Brief takes effect on the next recommendation cycle.
- The user can view and edit what the Brief holds about them at any time, and receives confirmation when their edits are saved.

Notes:
- The Brief is the single source of truth that anchors matching, ranking, and message drafting across all epics.
