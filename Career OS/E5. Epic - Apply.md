
# Epic: Apply

[[B. Value chain with objectives]]

---

## Objective

Raise application quality and referral hit rate while cutting time spent on logistics.

## JTBD

When I decide to apply, I want the right contact identified and a referral message ready, so I get a warm intro without the manual effort.

## User Stories

### 1. Find Referees via Affinity Match (Alumni / Ex-colleague)

As a job seeker, I want the platform to surface people at my target company who share my background — same school, past employer— so that I can reach out for a warm-ish referral even when I don't already know anyone there.

Affinity logic (drawn from the Candidate Brief):

Dimensions matched against the candidate:
- **Education** — same institute (e.g., IITB), with batch granularity:
  - same batch (same graduation year) — highest relevance
  - one batch senior (graduation year − 1)
  - one batch junior (graduation year + 1)
  - wider same-institute window (e.g., ± 3 batches) — lower relevance
- **Past employer** — shared previous company (ex-colleague, e.g., ex-ABInBev).
- **Location** — same city (e.g., Bangalore).

Target: people currently working at the company being applied to (e.g., Zepto).

Surface format: grouped by affinity reason, e.g. *Zepto × (ex-ABInBev, IITB, IIMA)*.

Ranking (most to least relevant; affinity boosts stack):
1. Same institute, same batch
2. Same institute, one batch above / one batch below
3. Same institute, wider batch window
4. Shared past employer

(A referee matching multiple dimensions — e.g., IITB same batch AND ex-ABInBev — ranks above a single-dimension match.)

Acceptance Criteria:
- For a given target company, the platform returns a ranked list of potential referees matching the candidate's affinity dimensions.
- Each referee card shows name, current role at the target company, and the matched affinity reason(s) (e.g., "IITB '19 · ex-ABInBev").
- Each card links out to the referee's LinkedIn profile; clicking opens LinkedIn so the user can connect / message there (MVP action). The drafted referral message (Story 2) is available to copy and paste.
- If no affinity matches are found, the card shows a clear empty state and lets the user proceed to apply directly via the role's original apply URL.

Integration:
- Extract relevant results from 3rd party API and to resolve LinkedIn profile URLs

### 2. Draft Referral Message

As a job seeker, I want a ready-to-send referral request drafted for me based on the role and the contact, so that I do not have to spend time writing it myself.

Acceptance Criteria:
- Message is personalized to the specific role and referee, drawing on the Candidate Brief (including the matched affinity, e.g., shared school / past employer).
- User can edit the draft.
- User can copy the draft with one tap and paste it into their platform of choice (e.g., LinkedIn, WhatsApp, email). The platform does NOT send the message — sending happens outside our platform.

Example draft (how the text should read):
> Hi {RefereeFirstName}, hope you're doing well! I noticed {Company} is hiring for a {RoleTitle} ({JobID/Location}) and it lines up closely with my background in {KeyStrength1} and {KeyStrength2}. We also share {AffinityReason} (e.g., IITB / ex-ABInBev). Would you be open to referring me, or pointing me to the right person on the team? Happy to share my CV and anything else useful. Thanks so much!

### 3. Application Tracking (Kanban-style board)

As a job seeker, I want my applications laid out on a board — like a storyboard with columns — that I can drag and drop cards across, so that I can see and update where every role stands at a glance without maintaining a manual spreadsheet.

Acceptance Criteria:
- Tracking is presented as a Kanban-style board: each application is a card, and columns represent stages (e.g., Shortlisted → Referral Sent → Applied → Interview → Offer → Rejected).
- The user advances a role by dragging its card from one column to the next (drag-and-drop); dropping a card updates its status.
- Cards are added to the board automatically when the user acts on a role in-platform (shortlists via right swipe, drafts/copies a referral message, or pastes a URL).
- Status the platform cannot detect on external sites is set by the user via drag-and-drop; status the platform does control is set automatically.
- The board gives a full-funnel view of the user's pipeline at a glance, with a count per column.

Note:
- "Automatic" tracking covers in-platform actions; because applications submitted on external job sites cannot be detected, the user advances those cards manually by dragging them across columns.
