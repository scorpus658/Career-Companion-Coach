## User Stories

### 1. Web App (responsive, mobile-first)

As a job seeker, I want the full platform experience to work seamlessly in my mobile browser, so that I can manage my job search on the go, including during commutes or spare moments at work.

Acceptance Criteria:
- The product is a responsive web app — there is no native iOS or Android app in scope. The MVP is delivered entirely through the browser.
- All core features are fully functional and optimized for mobile-browser viewports (and degrade gracefully on desktop).
- UI is optimized for touch interactions including swipe, tap, and scroll.
- Page load time meets a defined performance threshold on a standard 4G connection.

Note:
- Native iOS/Android apps are explicitly out of scope for the MVP.

---

### 2. WhatsApp Integration — Configure Notifications

As a job seeker, I want to receive job match notifications and application updates via WhatsApp, so that I stay on top of my search without having to open the app.

Acceptance Criteria:
- User can connect their WhatsApp number in settings.
- User can configure which notification types to receive (new matches, status changes, referral responses).
- Each notification includes a deep link back to the relevant item in the platform.

Integration:
- WhatsApp Business API (templated, opt-in messaging).
- TBD : We may need to build an MCP so that our agent reads off the platform and surfaces to whatsapp (scope TBD)

---

### 3. Amplitude ( I feel we should kill amplitude for MVP)

As the platform, I want all key user actions tracked in Amplitude, so that the team can monitor engagement, identify drop-off points, and make informed product decisions.

Acceptance Criteria:
- All core user actions (swipe, apply, referral sent, profile update) are instrumented as Amplitude events
- Key funnels (onboarding completion, first swipe, first application) are set up as Amplitude charts
- Event taxonomy is documented and shared with the team before release
