## Objective

## User Stories

### 1. Job Listings Ingestion (3rd-party listings API)

As the platform, I want to ingest job listings via a third-party listings API, so that I always have a comprehensive and current pool of roles to match against without scraping individual job boards directly.

Acceptance Criteria:
- Listings are ingested through a single third-party listings API  that aggregates across boards (only LinkedIn)
- New listings are pulled and indexed within a defined SLA i.e., once every day pulls all new jobs in last 3 days
- Each ingested listing stores the fields needed downstream: title, company, location, work mode, seniority, salary (if available), source, and original apply URL.

Integration:
- Third-party listings API — provider TBD. All board coverage (LinkedIn) is obtained through this single API, not via direct per-board connections.

---

### 2. Refresh the Status of the Openings

As the platform, I want to periodically check and update the status of tracked job listings, so that users are never matched to or shown roles that are no longer active.

Acceptance Criteria:
- Platform refreshes listing status from the listings API on a defined cadence i.e., sliding window of 11 days
- Closed or filled roles are automatically removed from all user queues and the daily stack.
- Users are notified (via their configured channel — see Cross-cutting) if a role they have saved is no longer active.

Integration:
- Use tool like browser harness for a cost prohibitive approach to fetch status against jobs
