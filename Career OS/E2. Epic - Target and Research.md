## Objective

Continuously track and refresh relevant open roles across job boards, so the platform always has a high-quality pool to match against.

## User Stories

### 1. Job Listings Ingestion

As the platform, I want to ingest job listings (via a third-party listings API for MVP), so that I always have a comprehensive and current pool of roles to match against without scraping individual job boards directly.

Acceptance Criteria:
- Listings are ingested through a single third-party listings API that aggregates across boards (only LinkedIn Jobs in MVP)
- New listings from the past 24hrs are pulled and indexed once every day
- Each ingested listing stores the Job description and extracts the necessary fields needed downstream: title, company, location, work mode, seniority, salary (if available), source, and original apply URL.

Integration:
- Third-party listings API - Mindcase. All board coverage (LinkedIn) is obtained through this single API, not via direct per-board connections.

---

### 2. Refresh the Status of the Openings

As the platform, I want to periodically check and update the status of all the job listings, so that users are never matched to or shown roles that are no longer active.

Acceptance Criteria:
- Platform refreshes listing status for all the jobs which have active status (that are still open) and have ageing <=14 days (ageing = current date - job posting date)
- Closed or filled roles are automatically removed from all user queues and the daily stack.
- Users are notified (via their configured channel — see Cross-cutting) if a role they have saved is no longer active.
